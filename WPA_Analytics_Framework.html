🤖 ML / AI Signals embedded across all roles
Predictive · Per Account
Propensity to Collect Score
Probability (0–1) that a claim will be collected without escalation. Drives queue prioritization and write-off risk flagging.
Inputs: balance, age, payer class, denial history, collector touch count, days since last activity, payer timer compliance
Output: score + recommended action tier (Work Now / Monitor / Escalate / Review for Write-off)
Predictive · Pre-Submission
Denial Prediction Score
Likelihood a claim will be denied before it is submitted or re-submitted. Enables proactive intervention.
Inputs: CARC/RARC history for this payer, claim type, service line, prior denial pattern, timely filing window
Output: denial risk % + top 3 likely denial codes + suggested fix
NLP · Note Quality
NLI Note Quality Score
4-dimension NLI scoring of flo_notes text. Continuous score 0–1 replacing binary notes_rate flag.
Inputs: flo_notes text → Action Documented, Next Step Specified, Denial Reason Referenced, Payer/Contact Named
Output: per-note score + dimension breakdown + coaching flag trigger
Detection · Workflow
Stall Prediction Model
Flags accounts likely to stall (no activity for 14+ days) before they stall, enabling preemptive reassignment.
Inputs: days since last touch, payer response rate, account age, collector workload, prior stall history
Output: stall risk score + estimated stall date + recommended reassign trigger
Forecasting · Staffing
Staffing Demand Forecast
90-day forward projection of required FTE by team based on AR volume trend, complexity, and attrition signals.
Inputs: AR volume 12-month rolling, claim complexity weights, current productivity rates, seasonal patterns
Output: FTE gap by week + hire/reduce recommendation + confidence interval
Detection · Anomaly
Payer Behavior Anomaly
Detects statistically unusual payer behavior — sudden denial rate spikes, payment delays, policy changes — vs. rolling baseline.
Inputs: payer-level denial rate, payment timing, CARC code distribution, reimbursement rate — all vs. 90-day rolling avg
Output: anomaly flag + deviation severity + suggested escalation path
Recommendation · Action
Next Best Action Engine
Per-account recommendation of the highest-ROI action a collector should take next, ranked by expected recovery value.
Inputs: claim status, denial code, payer rules, prior action history, time sensitivity, balance tier
Output: ranked action list + estimated recovery value + urgency score
Classification · Denial
Denial Root Cause Classifier
Automatically classifies denials into root cause buckets (eligibility, authorization, coding, timely filing, medical necessity, duplicate) from CARC/RARC codes + claim data.
Inputs: CARC code, RARC code, service line, claim type, payer, prior claim history
Output: root cause category + confidence + preventability flag (upstream fix vs. appeal)
Scoring · Productivity
Complexity-Adjusted Productivity
Normalizes raw claims/day against the complexity of each collector's assigned account mix so performance is fairly comparable.
Inputs: balance_wt × age_wt × payer_wt × denial_wt per account, normalized to team mean = 1.0
Output: adjusted productivity score + complexity load index per collector
Detection · Collector
Collector Behavior Anomaly
Flags unusual patterns in individual collector activity — sudden productivity drop, cherry-picking easy accounts, unusual touch patterns — vs. their own rolling baseline.
Inputs: touch count, account complexity, notes rate, resolution rate, time-of-day patterns — vs. 30-day personal baseline
Output: anomaly flag + severity + category (productivity / quality / pattern) + supervisor alert
